Parsanol is a modern parser combinator library that combines the elegance of PEG parsing with the performance of Rust. It provides multiple parsing backends, rich error reporting, and seamless Ruby bindings.

## Core Features

- **Packrat parser** with O(n) complexity guarantee
- **Bytecode VM** for fast execution
- **Arena allocation** for zero-copy AST construction
- **Rich error messages** with tree-structured context
- **Derive macros** for automatic AST conversion
- **Ruby FFI bindings** with 5 integration approaches

## Multiple Backends

| Backend | Use Case | Performance |
|---------|----------|-------------|
| Packrat | General purpose, left recursion | O(n) memoized |
| Bytecode | Production, speed-critical | Fastest execution |
| Auto | Best of both worlds | Automatic selection |

## Ruby Integration

Five integration approaches from pure Ruby to ZeroCopy+Slice for maximum flexibility and performance. Parslet-compatible API makes migration straightforward.

```ruby
# Parslet-compatible Ruby API
require 'parsanol'

class MyParser < Parsanol::Parser
  rule(:integer) { match('[0-9]').repeat(1) }
  rule(:plus)    { str('+') }
  rule(:expr)    { integer >> plus >> integer }

  root :expr
end

parser = MyParser.new
result = parser.parse('1+2')
```

### Integration Modes

| Mode | Method | Description |
|------|--------|-------------|
| Ruby | `parse_ruby` | Pure Ruby, maximum compatibility |
| Native | `parse_native` | Rust FFI, 17-42x faster |
| JSON | `parse_json` | Serialize result to JSON |
| Slice | `parse_slice` | Return Slice objects for lazy extraction |
| ZeroCopy | `parse_zerocopy` | Direct FFI pointer access |
