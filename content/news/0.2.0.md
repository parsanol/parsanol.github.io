Parsanol 0.2.0 brings streaming parser support, comprehensive benchmarking tools, and significant documentation improvements.

## Streaming Parser

Parse large files without loading them into memory. The streaming parser processes input in configurable chunks with a sliding window for backtracking support.

```rust
use parsanol::portable::streaming::{StreamingParser, ChunkConfig};

let config = ChunkConfig {
    chunk_size: 1024 * 1024, // 1MB chunks
    window_size: 2,           // 2 chunk lookback
};

let mut parser = StreamingParser::new(&grammar, config);
let result = parser.parse_from_reader(&mut file, &mut arena)?;
```

### ChunkConfig Options

| Option | Default | Description |
|--------|---------|-------------|
| `chunk_size` | 64KB | Bytes per read |
| `window_size` | 2 | Chunks kept for backtracking |
| `max_backtrack` | chunk_size × window_size | Maximum backtrack distance |

## Benchmark Suite

New comprehensive benchmark suite comparing all backends across different grammar types.

### Results vs Pure Ruby

| Grammar | Pure Ruby | parsanol-rs | Speedup |
|---------|-----------|-------------|---------|
| JSON | 45ms | 1.2ms | 37x |
| CSV | 120ms | 2.7ms | 44x |
| Calculator | 8ms | 0.4ms | 20x |

Run benchmarks:

```bash
# All examples
cargo bench --bench all-examples

# Backend comparison
cargo bench --bench backend-comparison
```
