Parsanol 0.2.0 introduces backend abstraction with trait-based selection, a new Bytecode VM backend for improved performance, and pre-commit hooks for local development.

## Backend Abstraction

Choose the right parsing backend for your grammar with trait-based selection.

```rust
use parsanol::portable::backend::{ParsingBackend, Backend};

// Auto-select the best backend for your grammar
let backend = Backend::default_for_grammar(&grammar);

// Or explicitly choose a backend
let backend = Backend::Packrat;  // Traditional memoization
let backend = Backend::Bytecode; // VM-based parsing
```

## Bytecode VM Backend

The new Bytecode VM backend compiles grammar to bytecode instructions for faster execution on linear patterns.

### Backend Comparison

| Backend | Best For | Performance |
|---------|----------|-------------|
| Packrat | General purpose, left recursion | O(n) memoized |
| Bytecode | Linear patterns, no nested repetition | Faster execution |
| Auto | Best of both worlds | Automatic selection |

## Pre-commit Hooks

CI checks now available locally:

```bash
# Install pre-commit hooks
pre-commit install

# Or use the shell script directly
./.githooks/pre-commit
```

Checks: format, clippy, docs, machete, typos, semver.

## What's Changed

- **Backend Abstraction**: `ParsingBackend` trait for custom backends
- **Bytecode VM**: VM-based parser for linear patterns
- `Backend::default_for_grammar()` for auto-selection
- Pre-commit hooks for local CI checks
- FFI module reorganization (`parsanol::ffi`)
- Breaking: `once_cell` feature removed (use `std::sync::OnceLock`)
